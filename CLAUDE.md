# CLAUDE.md

Guidance for Claude Code when working in this repository.

## What this is

**Play & Learn** — a single-file, offline HTML learning game for two young kids,
built to scale with their ages. It is spoken-first (the device reads prompts
aloud); kids tap pictures, no reading required except the Letters category.

- **Owen** — younger (~2.5), internal level name `simple`.
- **Lucas** — older (~4.5), internal level name `complex`.

## The one file that matters

Everything lives in **`Kids Learning Game - 2026-06-04.html`** (note the spaces
and date in the filename). It is fully self-contained:

- One `<style>` block (CSS), one `<script>` block (vanilla JS). No frameworks,
  no build step, no package manager, no external asset files.
- It's ~1 MB because animal-sound recordings are embedded as base64 data URIs.
- Runs by opening the file in any browser, fully offline. Also served via GitHub
  Pages (the published link the owner uses on their phone).

`README.md` is the human-facing overview. `docs/` holds product docs (e.g. the
Conversation Sparks PRD).

## Architecture (inside the HTML)

- **Screens:** `<section class="screen" id="...">` blocks; exactly one has
  `.active`. Switch with `showScreen(id)`. Screens: `#home`, `#play`, `#sparks`.
- **State:** a single `state` object (`level`, `category`, `stars`, `current`, …).
- **Categories:** the `CATS` object; each entry has `label`, `icon`, `color`,
  optional `lucasOnly`, and a `build()` that returns a round. Home tiles are
  generated by `buildHome()`. `allowed()` hides `lucasOnly` categories for Owen.
- **Speech:** `speak(text)` (Web Speech API), with the voice chosen in
  `loadVoice()`. Audio is unlocked by the **Start** gate on first tap.
- **Sound effects:** generated with the Web Audio API (`tone()`); animal sounds
  are the embedded recordings (`playAnimal()`).
- **Conversation Sparks:** parent tool on `#sparks`. One read-aloud question at a
  time from the `SPARKS` array; tap-to-speak (`speak()` only on the Listen
  button), `Next`/card advances. Deck is shuffled on entry, no repeats until
  exhausted, then reshuffled. Entered via the low-key sticky button at the bottom
  of `#home`.

## Conventions / gotchas

- **Keep it one self-contained file.** No external assets, no dependencies, must
  work offline. Inline new CSS/JS into the existing blocks.
- **Guard speech APIs:** wrap direct `speechSynthesis.cancel()` calls in
  `if ('speechSynthesis' in window)`; `speak()` already guards internally.
- **Difficulty toggle UX:** a plain tap switches Owen/Lucas on the home screen,
  but mid-game it requires a press-and-hold so a kid can't flip it by accident.
- **No reading required** for kids except the Letters category — new content
  should be spoken or pictorial.
- Match the existing terse, single-line CSS style and the existing JS idioms
  (helpers `$`, `rand`, `shuffle`, `sample`, `pick`).

## Testing a change

No test suite. Verify behavior by driving the file in a headless browser
(Playwright/Chromium is available in this environment). Typical smoke test:
load the file, click `#startBtn` to pass the audio gate, then exercise the
screen/flow you changed and assert on DOM text + check for console errors.

## Git / workflow

- Work on a feature branch, push, open a PR, merge into `main` (don't commit
  straight to `main`).
- `main` is what GitHub Pages publishes; after merging, Pages takes a few minutes
  to redeploy and caches aggressively (use a `?v=N` query string to bust the
  iOS/CDN cache when verifying on a phone).
